CSS Variables

You can define variables in CSS, they can be defined at a global level as well as in specific elements. You can also define fallback values.

Usage:
:root {
  --hue: 220;
  --saturation: 60%;
  --lightness: 70%;
  --main-color: hsl(var(--hue) var(--saturation) var(--lightness));
}

.text {
  color: var(--main-color, black);
}

.custom-hue {
  --hue: 100;
  --main-color: hsl(var(--hue) var(--saturation) var(--lightness));
  color: var(--main-color, black);
}

CSS Houdini

CSS Houdini is a set of APIs that expose parts of the CSS engine. This makes it easier for developers to create extensions for CSS. These extensions might be to polyfill features that are not yet available in a browser, experiment with new ways of doing layout, or add creative borders or other effects.

Registering a new property can make it animatable (if syntax type is animatable).

Usage:
/* in CSS */
@property --color-stop {
  syntax: '<color>';
  inherits: false;
  initial-value: transparent;
}

/* or, in JS */
CSS.registerProperty({
  name: '--color-stop',
  syntax: '<color>',
  inherits: false,
  initialValue: 'transparent'
});

Advanced @media

prefers-color-scheme

The prefers-color-scheme is used to detect if the user has requested a light or dark color theme.

With media query
Without media query

Usage:
@media (prefers-color-scheme: dark) {
  .section { background: #333; color: white; }
}

@media (prefers-color-scheme: light) {
  .section { background: #eee; color: black; }
}

Resources:
developer.mozilla.org

prefers-reduced-motion

The prefers-reduced-motion is used to detect if the user has requested that the system minimize the amount of non-essential motion it uses.

Usage:
@media (prefers-reduced-motion: no-preference) {
  .pulse { animation-name: pulse-animation; }
}

@media (prefers-reduced-motion: reduce) {
  .pulse { animation-name: none;
}

Resources:
developer.mozilla.org

prefers-reduced-data

The prefers-reduced-motion is used to detect if the user has requested the web content that consumes less internet traffic.

Usage:
<link rel="preload" href="font.woff2" as="font" media="(prefers-reduced-data: no-preference)" crossorigin>

@media (prefers-reduced-data: no-preference) {
  .background { background-image: url(image4k.png); }
}

@media (prefers-reduced-data: reduce) {
  .background { background-image: url(image720.png); }
}

Resources:
developer.mozilla.org

color-gamut

The color-gamut can be used to test the approximate range of colors that are supported by the user agent and the output device.

Usage:
@media (color-gamut: srgb) {
  .background { background-color: #00FF00; }
}

@media (color-gamut: p3) {
  .background { color(display-p3 0 1 0); }
}

@media (color-gamut: rec2020) {
  .background { color(rec2020 0 1 0); }
}

Nesting module

Nesting module, @nest

To be nest-prefixed, a nesting selector must be the first simple selector in the first compound selector of the selector. If the selector is a list of selectors, every complex selector in the list must be nest-prefixed for the selector as a whole to nest-prefixed.
- csswg.org

&

Usage:
.foo {
  color: blue;
  & > .bar { color: red; }
}
/* equivalent to
  .foo { color: blue; }
  .foo > .bar { color: red; }
*/

figure {
  margin: 0;

  & > figcaption {
    background: hsl(0 0% 0% / 50%);

    & > p {
      font-size: .9rem;
    }
  }
}
/* equivalent to
  figure { margin: 0; }
  figure > figcaption { background: hsl(0 0% 0% / 50%); }
  figure > figcaption > p { font-size: .9rem; }
*/

Resources:
csswg.org

@nest

Usage:
.foo {
  color: red;
  @nest .parent & {
    color: blue;
  }
}
/* equivalent to
  .foo { color: red; }
  .parent .foo { color: blue; }
*/

.foo {
  color: red;
  @nest :not(&) {
     color: blue;
  }
}
/* equivalent to
  .foo { color: red; }
  :not(.foo) { color: blue; }
*/

Resources:
csswg.org

:is, :where

The :is() and :where() functions takes a selector list as its argument, and selects any element that can be selected by one of the selectors in that list. This is useful for writing large selectors in a more compact form.

The difference between :where() and :is() is that :where() always has 0 specificity, whereas :is() takes on the specificity of the most specific selector in its arguments.

Usage:
header p:hover,
main p:hover,
footer p:hover {
  color: red;
  cursor: pointer;
}

:where(header, main, footer) p:hover {
  color: red;
  cursor: pointer;
}
